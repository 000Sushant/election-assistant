# 🗳️ Votika: AI-Powered Election Guide Assistant

Votika is a sophisticated, AI-driven assistant designed to seamlessly guide Indian voters through the complexities of the electoral process. Built with a focus on absolute simplicity, speed, accessibility, and high performance, Votika serves as a mentor and guide for upcoming elections.

![Votika UI](https://via.placeholder.com/800x400?text=Votika+Dashboard+Preview)

## ✨ Key Features

- **🧠 Intelligent Persona**: Powered by **Gemini 1.5 Flash**, Votika communicates as an articulate mentor using extremely simple, accessible English to ensure anyone can understand the electoral process.
- **🗣️ Hindi Audio Capabilities**: Features a "Listen in Hindi" function utilizing **Google Cloud Translation** and **Google Cloud Text-to-Speech** (Neural Voices) to cater to diverse linguistic needs.
- **📅 Dynamic Election Dashboard**: Real-time tracking of upcoming Indian elections with automated refresh cycles dynamically generated by Gemini.
- **♿ Fully Accessible**: The frontend UI is built with robust `aria-label` tags, fully supporting screen-reading technology for visually impaired citizens.
- **📦 Dual-Layer Caching**: High-efficiency architecture utilizing both **In-Memory** and **Google Cloud Firestore Native Mode** caching to minimize API costs and latency.
- **🚀 Cloud Run Ready**: Fully containerized using a multi-stage Docker build, serving an optimized, GZIP-compressed Angular frontend seamlessly through an Express backend.

## 🛠️ Tech Stack & Google Services

- **Frontend**: Angular 18+, SCSS (Accessible, Modular architecture)
- **Backend**: Node.js, Express.js (GZIP Compressed, Helmet Protected)
- **Containerization**: Docker (Multi-stage build)
- **Google Cloud Services**:
  - Vertex AI (Gemini 1.5 Flash)
  - Cloud Firestore (Native Mode)
  - Cloud Translation API
  - Cloud Text-to-Speech API
  - Application Default Credentials (ADC) Integration

## 🚀 Getting Started Locally

### Prerequisites
- Node.js (v20+)
- A Google Cloud Project with the following APIs enabled:
  - Vertex AI API
  - Cloud Firestore API
  - Cloud Translation API
  - Cloud Text-to-Speech API

### Local Development Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/election-assistant.git
   cd election-assistant
   ```

2. **Frontend Setup**
   ```bash
   cd frontend
   npm install
   npm start # Runs local dev server on port 4200
   ```

3. **Backend Setup**
   ```bash
   cd ../backend
   npm install
   ```

4. **Configuration**
   Create a `.env` file in the `backend` directory:
   ```env
   PROJECT_ID=your-google-cloud-project-id
   VERTEX_REGION=asia-south1
   MODEL_NAME=gemini-1.5-flash-001
   GOOGLE_APPLICATION_CREDENTIALS=service-account.json
   PORT=3000
   ```
   *(Ensure your `service-account.json` is placed in the `backend` folder. This file is ignored by Git for security.)*

5. **Start Backend Server**
   ```bash
   npm start
   ```

## 🐳 Deployment (Google Cloud Run)

This repository is optimized for deployment on Google Cloud Run as a single cohesive container.

1. **Build the Docker Image**
   ```bash
   docker build -t votika-app .
   ```
   *The multi-stage `Dockerfile` will automatically compile the production Angular app and serve it directly from the Express backend.*

2. **Deploy to Cloud Run**
   When deploying to Cloud Run, Votika automatically detects the environment and falls back to **Google Application Default Credentials (ADC)**. You do not need to configure any secret JSON files in production.

- **Service-Oriented Architecture**: All business logic and AI orchestration are handled within the `ElectionService`.
- **IT Standards Aligned**:

  - **Efficiency**: GZIP compression, multi-stage lightweight Docker images.
  - **Code Quality**: Relative API routing, strict separation of concerns.
